You have a device that uses Seven-segment display to display 2 digit numbers. However, some of the segments are out of order and can't display.

You will be given information on lit and broken segments. You don't know whether each broken segment is lit or not. You have to count and return the total number that the device may be displaying.

The input is a set of lit segments(first argument) and a set of broken segments(second argument).

  • Upper case letters represent tens place segments
  • Lower case letters represent ones place segments
  • top most: 'A(a)', top right: 'B(b)', bottom right: 'C(c)', bottom most: 'D(d)', bottom left: 'E(e)', top left: 'F(f)', middle: 'G(g)'

Example:

seven_segment({'B', 'C', 'b', 'c'}, {'A'}) == 2    #11, 71
seven_segment({'B', 'C', 'a', 'c', 'd', 'f', 'g'}, {'A', 'D', 'G', 'e'}) == 6    #15, 16, 35, 36, 75, 76
seven_segment({'A', 'B', 'C', 'D', 'E', 'F', 'a', 'b', 'c', 'd', 'e', 'f'}, {'G', 'g'}) == 4    #0, 8, 80, 88

Input: Two arguments. The first one is the lit segments as a set of alphabets representing segments. The second one is the broken segments as a set of alphabets representing segments.

Output: The total number that the device may be displaying.

Precondition:

  • all(re.match('[A-Ga-g]', s) for s in lit | broken)
  • len(lit & broken) == 0
